Hexo小技巧

发表于2022-11-29|更新于2022-12-09|建站
|字数总计:600|阅读时长:2分钟|阅读量:

  1. 建议在 blog/source/ 下创建诸如 img/css/js 等文件夹，专门存放文章或网站用的素材，分门别类后续也方便维护，这是个好习惯。

  2. hexo generate 打包的时候，会自动把 source/ 下的文件，包含到public/网站静态文件中去。当执行 hexo deploy 时，Hexo 会将 public 目录中的文件和目录推送至 _config.yml 中指定的远端仓库和分支中，并且完全覆盖该分支下的已有内容。

  3. 想要优雅地魔改主题，就不要改动源代码，而是创建新的CSS JS文件，通过引入的方式使用。这样，就不用担心主题升级导致魔改失效。另外，最好将自定义CSS JS放在Hexo根目录下，而不是主题的source/目录下

    通过主题配置 inject 可以引入外部的自定义 CSS 文件和 JS 文件。(全站，如果是某一页的，可以在该页里面直接引入。css文件在head引入，js文件在bottom里引入，有的js要求在head引入，不然无法生效)

  4. 想在Hexo一篇文章指定多个分类，要加上[]，如：

    如果不加[ ],会使B分类成为A分类的子分类，因为Hexo不支持多个同级分类！

  6. Butterfly支持 font-awesome v6图标。书写格式 图标名：url || 描述性文字。设置社交图标

  7. 由于Hexo 4.1开始，默认对网址进行解码，以至于如果是中文网址，会被解码，可设置decode: true来显示中文网址。

  8. 可开启图片Figcaption描述文字显示。优先显示图片的 title 属性，然后是 alt 属性。修改 主题配置文件 photofigcaption: true
