什么是Hexo?

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

环境准备

请确保计算机中已安装了以下程序：

  1. Node.js
  2. Git

tips：Node.js默认集成了npm

Hexo安装

全局安装Hexo脚手架，安装后可以执行一系列Hexo命令
当你的电脑上有多个博客项目时，不需要重复安装

Hexo初始化

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件，并进行初始化，交给Hexo管理

初始化完成后，/myblog/文件夹的目录结构如下：

1.scaffolds
当您新建文章时，Hexo会根据 scaffold来建立文件。
Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。
2.source
资源文件夹是存放用户资源的地方。
生成静态文件时，Hexo会读取source目录下的内容，打包到public目录中
除 _posts 文件夹之外，开头命名为 _ (下划线)的文件或文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
3.themes
主题文件夹。
Hexo 会根据主题来生成静态页面。

应用主题

修改 Hexo 根目录下的 _config.yml，把主题改为butterfly

为了减少主题维护或升级主题后带来的不便，请使用以下方法（可选）：

在 hexo 的根目录创建一个文件 _config.butterfly.yml，并把主题目录的 _config.yml 内容复制到 _config.butterfly.yml 去。( 注意: 复制的是主题的 _config.yml ,而不是 hexo 的 _config.yml)(注意： 不要把主题目录的 _config.yml 删掉)

📢 复制的是主题目录/theme/butterfly下的 `_config.yml`

📢 Hexo会自动合併主题中的/theme/butterfly/_config.yml和 _config.butterfly.yml里的配置，如果存在同名配置，会优先使用_config.butterfly.yml中的配置

Gitee部署

具体部署过程请参考Hexo-零基础搭建个人博客(详解)这里说几个注意点：

  1. 仓库必须是公开的
  2. 仓库名称要与gitee用户名一致（固定写法username.gitee.io）
  3. 若是后期修改了自己的代码，需要到gitee上手动点击更新，新的内容才会生效

tips：部署成功后，如果网站只显示文字，样式丢失，原因可能是：

url错误

检查根目录下配置文件_config.yml中网站地址url,填写Gitee Pages上生成的链接

浏览器缓存

清除缓存后再次访问
